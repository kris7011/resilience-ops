# Project Goals

## Primary goal

Build a complete fullstack SaaS application using Angular, ASP.NET Core and Microsoft Azure.

The project should demonstrate the complete development lifecycle:

1. Repository creation
2. Local project setup
3. Frontend and backend development
4. Database integration
5. Automated testing
6. Logging and observability
7. Continuous integration
8. Cloud deployment
9. Production monitoring
10. Technical documentation

## Product goal

ResilienceOps should help organizations register, assess and follow up on operational risks and incidents.

The application should provide a clear overview of:

- Current risks
- Risk severity
- Responsible owners
- Mitigation actions
- Deadlines
- Reported incidents
- Changes made to important records

## Development principles

### Understand before proceeding

Every significant piece of code should be understood before the next feature is added.

### Small changes

Each commit should represent one logical and explainable change.

### Clear responsibility

Every class, component and function should have a clear responsibility.

### Testability

Business logic should be testable without requiring the full application to run.

### Appropriate complexity

Patterns and abstractions should only be introduced when they solve a concrete problem.

### Documentation

Important choices, alternatives and consequences should be documented.

## Definition of success

The project is successful when:

- The application can run locally.
- The frontend communicates with the backend.
- Data is stored in a relational database.
- Core business features are implemented.
- Automated tests run successfully.
- The application is deployed to Azure.
- Application Insights receives telemetry.
- GitHub Actions builds, tests and deploys the application.
- The project can be demonstrated from a public URL.
- Every important architectural and technical choice can be explained.
